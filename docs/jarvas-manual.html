Jarvas

Manual de instruções — assistente de IA distribuído com Managed Agents e suporte a modelos locais via Ollama

Python 3.11+ FastAPI OpenRouter Gemini SDK VSCode Extension Ollama / Gemma 4

Visão Geral

O Jarvas é um assistente de IA local com múltiplos agentes especializados. Pode ser usado como REPL interativo no terminal ou como servidor de API com Managed Agents compatível com o padrão Anthropic.

Com a versão 0.4, o Jarvas agora inclui o agente gemma-local que roda completamente offline via Ollama, com ferramentas nativas do VSCode — podendo abrir, editar e criar qualquer arquivo do workspace diretamente pelo chat.

✅ Modo 100% offline
Para usar apenas o agente gemma-local via Ollama, você não precisa de nenhuma API key. Instale o Ollama, baixe o modelo e inicie o servidor — tudo roda localmente.

Instalar Ollama Novo

O Ollama é o runtime que baixa e executa modelos de linguagem localmente. Ele expõe uma API compatível com OpenAI em localhost:11434.

Windows

# Via winget (recomendado)
winget install Ollama.Ollama

# Após instalar, verificar se está rodando:
curl http://localhost:11434/api/tags

Linux / macOS

curl -fsSL https://ollama.com/install.sh | sh
ℹ️ Ollama como serviço
No Windows, o Ollama é instalado como serviço e inicia automaticamente com o sistema. No Linux/macOS, use ollama serve para iniciar manualmente.

Baixar Gemma 4 Novo

O Jarvas usa Gemma 4 por padrão. Escolha a variante conforme sua memória de GPU (VRAM):

ModeloVRAM mínimaComando
gemma4 (12B Q4) 8 GB ollama pull gemma4
gemma3:27b 16 GB ollama pull gemma3:27b
gemma3:4b (leve) 4 GB (ou só CPU) ollama pull gemma3:4b 
# Baixar o modelo padrão
ollama pull gemma4

# Listar modelos instalados
ollama list

# Testar interativamente
ollama run gemma4 "Olá, você consegue me ajudar com Python?"
⚠️ Uso sem GPU
Se não tiver GPU dedicada, o modelo roda na CPU — muito mais lento. Recomendamos gemma3:4b para CPUs. O download pode demorar: o Gemma 4 (12B) tem ~8 GB.

Usar outro modelo no Jarvas

Ao criar ou atualizar um agente via API, defina o campo model com o prefixo ollama/:

# Exemplos de model IDs suportados
"ollama/gemma4"
"ollama/gemma3:27b"
"ollama/llama3.2"
"ollama/mistral"
"ollama/phi4"

Agente gemma-local Novo

O agente gemma-local é pré-registrado automaticamente quando o servidor inicia. Ele é o único agente com acesso às ferramentas nativas do VSCode.

Iniciar tudo (ordem correta)

  1. Garantir que o Ollama está rodando 
    # Windows: já roda como serviço após instalação
# Linux/macOS:
ollama serve

# Verificar:
curl http://localhost:11434/api/tags
  2. Iniciar o servidor Jarvas 
    jarvas --managed
# Servidor disponível em http://localhost:8000
  3. Verificar agentes disponíveis 
    curl http://localhost:8000/v1/agents
# Deve retornar: hermes, gemini-guard, deepseek-guard, gemma-local
  4. Abrir o VSCode e usar o chat

    Clique no botão 🤖 Jarvas na barra de status inferior direita do VSCode. Selecione gemma-local no dropdown e comece a conversar.

Ferramentas Nativas do VSCode Novo

Quando o agente gemma-local (ou qualquer agente com essas tools configuradas) precisa interagir com o VSCode, a extensão executa a operação diretamente usando a API do VSCode e devolve o resultado ao agente. O servidor FastAPI pausa o loop e aguarda a resposta da extensão por até 60 segundos.

ℹ️ Como funciona o split tool execution
Ferramentas como bash, read e web_search rodam no servidor Python. Ferramentas vscode_* são delegadas à extensão VSCode via SSE — a extensão executa e devolve o resultado via POST /v1/sessions/{id}/tool_result.
vscode_open
Abre um arquivo no editor (nova aba)
path: string
vscode_edit
Edita um arquivo substituindo old_text por new_text (primeira ocorrência)
path, old_text, new_text: string
vscode_terminal
Envia um comando para o terminal integrado do VSCode
command: string
vscode_list
Lista arquivos do workspace por padrão glob (máx 200)
pattern?: string (ex: **/*.py)

Também disponíveis para todos os agentes

bash
Executa comando shell no servidor (timeout 30s)
command: string
read
Lê conteúdo de arquivo no servidor (máx 50KB)
path: string
write
Escreve arquivo no servidor
path, content: string
web_search
Busca na web via Gemini (requer GEMINI_API_KEY)
query: string

💬 No chat do VSCode, com gemma-local selecionado:

Você: "Abra o arquivo jarvas/managed/runtime.py e me mostre as funções de dispatch"

→ Gemma usa vscode_open → arquivo abre na aba ao lado

Debate Multi-Agente — /debate

Inicia um debate estruturado entre Gemini e DeepSeek. O orquestrador apresenta a questão, coleta as perspectivas e sintetiza as conclusões.

/debate Qual é a melhor forma de estruturar uma API REST em Python?
/debate Vale mais a pena usar SQL ou NoSQL para um app de notas?
ℹ️ Fluxo do debate
Orquestrador → Gemini (perspectiva A) → DeepSeek (perspectiva B) → Orquestrador (síntese)

Iniciar o Servidor de Managed Agents

jarvas --managed

O servidor FastAPI sobe em http://localhost:8000. Documentação Swagger disponível em http://localhost:8000/docs.

Porta personalizada

jarvas --managed --port 3000

Muda a porta do servidor para 3000 (ou qualquer outro valor que você escolher).

ℹ️ Portas personalizadas

Você pode usar qualquer porta disponível entre 1024 e 65535. Isso é útil quando:

  • A porta padrão 8000 já está em uso por outro serviço
  • Seu firewall ou rede corporativa bloqueia a porta 8000
  • Você quer executar múltiplas instâncias do Jarvas simultaneamente
  • Quer manter projetos organizados em portas específicas
✅ Verificar portas em uso

Para verificar se uma porta já está sendo usada:

# Windows
netstat -ano | findstr :8000

# Linux / macOS
lsof -i :8000

Criar agente com Ollama

curl -X POST http://localhost:8000/v1/agents \
  -H "Content-Type: application/json" \
  -d '{
    "name": "meu-agente-local",
    "model": "ollama/gemma4",
    "system_prompt": "Você é um assistente de código especializado em Python.",
    "tools": ["read", "bash", "vscode_open", "vscode_edit"]
  }'

Atualizar agente (com version check)

curl -X PATCH http://localhost:8000/v1/agents/<ID> \
  -H "Content-Type: application/json" \
  -d '{
    "version": 1,
    "tools": ["read", "web_search", "vscode_edit"]
  }'
# Retorna 409 se version não coincidir com a atual

Fluxo completo via curl

# 1. Criar sessão
SESSION=$(curl -s -X POST http://localhost:8000/v1/sessions \
  -H "Content-Type: application/json" \
  -d '{"agent_id": "<GEMMA_ID>"}' | python -c "import sys,json; print(json.load(sys.stdin)['id'])")

# 2. Enviar mensagem
curl -X POST http://localhost:8000/v1/sessions/$SESSION/events \
  -H "Content-Type: application/json" \
  -d '{"content": "Liste os arquivos .py do projeto"}'

# 3. Ouvir o stream SSE
curl -N http://localhost:8000/v1/sessions/$SESSION/stream

Eventos SSE

Todos os eventos chegam no formato data: {JSON}\n\n. Cada evento tem sempre os campos type, session_id e timestamp.

user.message
Mensagem do usuário persistida na sessão. Campo: content
agent.message
Resposta final do agente. Campos: content, model
agent.tool_use
Agente chamou uma tool. Campos: tool_name, tool_input, tool_call_id, awaiting_callback
agent.tool_result
Resultado de uma tool. Campos: tool_name, tool_call_id, output, is_error
session.status_idle
Sessão concluída com sucesso. Sinal de fim de stream.
session.error
Erro na sessão. Campo: message
ℹ️ Heartbeat e Replay
O stream envia : heartbeat a cada 30s para manter a conexão viva (ignorado pelos clientes). Se você conectar ao stream após a sessão já ter terminado, os eventos são reproduzidos em replay automaticamente.

Campo awaiting_callback (ferramentas VSCode)

Quando o agente gemma-local chama uma vscode_* tool, o evento agent.tool_use inclui "awaiting_callback": true. A extensão VSCode executa a operação e chama POST /v1/sessions/{id}/tool_result com o resultado. O servidor então continua o loop do agente.

A extensão adiciona um botão 🤖 Jarvas na barra de status inferior direita do VSCode que abre o painel de chat.

    npm run compile

Usar o Chat no VSCode

  1. Certifique-se que o servidor está rodando 
    jarvas --managed
  2. Clique no botão na status bar

    O botão 🤖 Jarvas fica no canto inferior direito. Um painel de chat abre ao lado do editor.

  3. Selecione o agente no dropdown

    Escolha gemma-local para edição de arquivos offline, ou outro agente para tarefas específicas.

  4. Converse normalmente

    O agente tem acesso às ferramentas do VSCode. Você verá as chamadas de ferramenta exibidas de forma colapsável no chat (🔧 vscode_edit).

✅ Dicas de uso com gemma-local
  • Diga ao agente o caminho completo dos arquivos, ou peça para ele listar com vscode_list primeiro
  • Para edições precisas, peça ao agente para ler o arquivo antes de editar
  • O vscode_terminal envia o comando mas não espera pelo resultado — use para iniciar processos
  • O agente pode alternar entre read (server-side) e vscode_open (VSCode-side) dependendo da task

Agentes Pré-configurados

hermes

Agente orquestrador principal. Usa nousresearch/hermes-3-llama-3.1-70b via OpenRouter. Pode invocar gemini-guard e deepseek-guard como sub-agentes via a tool call_agent.

Tools: bash, write, read, web_search

gemini-guard

Especializado em busca na web e processamento de informações. Usa gemini-2.5-flash diretamente via Google SDK.

Tools: web_search, read

deepseek-guard

Especializado em análise técnica e escrita de código. Usa deepseek/deepseek-chat via OpenRouter.

Tools: read, write

gemma-local Novo

Agente autônomo para edição de código no VSCode. Roda 100% offline via Ollama (gemma4). Único agente com ferramentas nativas do VSCode.

Tools: read, web_search, vscode_open, vscode_edit, vscode_terminal, vscode_list

ℹ️ Criar agentes customizados
Use POST /v1/agents para criar seus próprios agentes com qualquer combinação de modelo, tools e system prompt. Adicione skills para injetar contexto de domínio sem mudar o agente base.

Solução de Problemas

Ollama não responde

# Verificar se está rodando
curl http://localhost:11434/api/tags

# Windows: verificar o serviço
Get-Service Ollama

# Reiniciar
ollama serve

Agente gemma-local trava (tool timeout)

O agente pausa por até 60 segundos esperando a extensão VSCode executar uma tool. Se o chat não responder, verifique se a extensão está ativa (botão aparece na status bar) e o servidor está em execução.

Erro 409 no PATCH de agente

O campo version no body não corresponde à versão atual do agente. Busque o agente com GET /v1/agents/{id} para obter a versão atual e tente novamente.

SSE stream encerra imediatamente

A sessão já estava no estado idle quando você conectou. O servidor envia os eventos em replay e encerra. Isso é comportamento esperado — os dados estão todos no replay.

Extensão VSCode não aparece

# Verificar se compilou corretamente
cd jarvas-vscode
npm run compile

# Em modo de desenvolvimento: F5 no VSCode para abrir Extension Development Host

web_search falha

A busca web usa o Gemini. Certifique-se que GEMINI_API_KEY está configurada no .env mesmo quando usar o agente gemma-local (que roda localmente, mas delega a busca ao guard Gemini).
