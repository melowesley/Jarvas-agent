01 · Visão geral

O que é o Jarvas?

O Jarvas é uma plataforma de IA conversacional com roteamento adaptativo de tarefas. Em vez de depender de um único modelo ou modo de operação, ele classifica cada mensagem, seleciona o agente mais adequado e orquestra o fluxo — com memória, ferramentas e observabilidade integradas.

A interface de uso é tripla: CLI interativo para uso direto no terminal, modo gerenciado com UI web e API REST (porta 8000), e integração via SDK para acoplamento em outros produtos.

Diferencial de v0.5.0: a arquitetura deixou de ser um único handler centralizado e passou a ser um supervisor formal que registra, roteia e monitora agentes especializados — com fallback gracioso para os handlers originais caso algum módulo não esteja disponível.

02 · Avanços na v0.5.0

O que mudou nesta versão

Área Avanço
Multiagente novo Supervisor central (agents/supervisor.py), registry unificado, protocolos formais de agente e estratégias de roteamento.
Toolset seguro atualizado Ferramentas com schemas explícitos, confirmações de ambiente e controle de escopo — reduzindo riscos de operações não intencionais.
Observabilidade novo Dump de sessão, logs estruturados e rastreabilidade de rota — facilitando diagnóstico e auditoria.
Memória enriquecida atualizado MemPalace com metadados de sessão mais ricos: contexto de projeto, última tarefa, tipo de intent e histórico de agentes.
Resiliência atualizado Fallback automático para handlers individuais quando o Supervisor não está disponível — sem perda de funcionalidade.
03 · Agentes e Guardas
🧠
Hermes (LLM Principal)
Modelo principal de conversação, raciocínio e geração de texto. Orquestra a sessão e delega para guardas especializados.
🔎
Gemini Guard
Análise, síntese e busca web. Especializado em tarefas que exigem pesquisa ou sumarização de grandes volumes.
⚙️
DeepSeek Guard
Estruturação, código e organização lógica. Usado em tarefas de engenharia e scaffolding de arquivos.
🗄️
MemPalace
Armazenamento e recuperação de contexto entre sessões. Registra aprendizados, preferências e histórico de projeto.
04 · Integração com AgendaVet

Como o Jarvas se conecta ao AgendaVet

O Jarvas não é um produto vertical — é uma camada de inteligência acoplável. No contexto do AgendaVet, ele pode atuar como motor de processamento de linguagem natural para atendimento automatizado, triagem de solicitações, extração de intenção e geração de respostas contextualizadas para tutores e clínicas.

AgendaVet (Produto)
→
API REST / SDK Jarvas
→
Supervisor + Agentes
→
MemPalace + Supabase

A base v0.5.0 está tecnicamente pronta para essa integração. A principal ressalva é que partes do código ainda carregam nomenclatura de domínio legada (ex: autoescola) que deve ser removida ou isolada antes de uma entrega formal ao produto.

Nota: a decisão sobre renomear o domínio legado ou mantê-lo em task isolada é a única pendência de produto antes de usar o Jarvas como base do AgendaVet.
