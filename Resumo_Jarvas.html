Projeto Jarvas

Resumo Executivo do Sistema — Versão 0.3.0

1. O que é o Jarvas?

O Jarvas é um assistente de inteligência artificial distribuído. Diferente de um chatbot comum que utiliza apenas um modelo, o Jarvas atua como um maestro (orquestrador). Ele analisa o que o usuário está pedindo e roteia automaticamente a tarefa para o modelo mais qualificado (como Claude para análises ou GPT-4o para visão).

Diferencial: O sistema incorpora "Guardas" — modelos especializados chamados para tarefas estruturais ou semânticas sob demanda, além de possuir suporte nativo para memória persistente e agentes instanciados.

2. Como posso interagir com ele?

O Jarvas (v0.3.0) oferece três vias de comunicação principais:

  • Terminal (CLI): Uma interface rica de linha de comando onde você conversa diretamente (REPL). Suporta slash commands para acesso rápido a recursos avançados.
  • Interface Web (UI): Um chat via navegador. Quando o Jarvas é rodado no modo gerenciado (jarvas --managed), ele sobe uma página HTML interativa na porta 8000.
  • Integração de Sistemas (API REST): Um backend completo em FastAPI para conectar o Jarvas a outros softwares, criar e listar agentes isolados (sessões) e acessar os guardas via HTTP.

3. A Força dos Guardas

O sistema conta com dois "Guardas" que operam de forma paralela ao chat principal, acionados por comandos ou requisições API:

Guarda Modelo Base Especialidade Uso Rápido (CLI)
Gemini gemini-2.5-flash Análise semântica rápida, busca na web (web search nativa) e síntese de informações. /g [msg] ou /g web [msg]
DeepSeek deepseek-chat Estruturação arquivística, organização de dados e formatação complexa. /d [msg] ou /d web [msg]

Nota: O Jarvas possui um comando incrível chamado /debate [tópico] que coloca o Guarda Gemini e o Guarda DeepSeek para debaterem sobre um assunto em rodadas estipuladas, gerando um consenso final valioso.

4. Memória e Persistência

O Jarvas nunca esquece (a menos que você queira):

  • MemPalace: Um sistema próprio de "Palácio da Memória" acessível via comandos /hmem. Permite guardar fragmentos de conhecimento, buscar semanticamente e gerar Grafos de Conhecimento (Knowledge Graphs).
  • Banco de Dados (Supabase): Todas as conversas, logs de guardas e resultados de debates são persistidos de forma assíncrona. Se algo der errado no banco, o Jarvas silencia o erro e não interrompe a sua conversa.

5. Novidades da Versão 0.3.0

  • Implementação do Modo Gerenciado (Managed Mode) com servidor Uvicorn/FastAPI via jarvas --managed.
  • Suporte nativo para criar, listar e conversar com Múltiplos Agentes usando os comandos /session.
  • Atualização do motor do Google para gemini-2.5-flash.
